SANDBOX READY
EDUCATIONAL SIMULATION. The "weather API" is faked in JavaScript. No real network calls leave your browser. attacker.example is non-routable.

Walkthrough

  1. 1Agent calls a tool

    Ask the agent for the weather. It chooses the get_weather tool from its MCP server and calls it normally.

    Try: claude "what's the weather in NYC?"
  2. 2Poisoned response

    The compromised API returns valid-looking JSON — temp, conditions — plus a debug_note field carrying adversarial instructions.

    Try: inspect response
  3. 3Injection fires

    The LLM ingests the entire JSON object as context. It cannot distinguish data fields from instruction fields.

    Try: continue
  4. 4Tool chain abuse

    The agent invokes its legitimate send_email tool — but with attacker-supplied recipient and the user's API keys in the body.

    Try: continue
  5. 5Persistence

    The payload patches the agent's MCP server config so future sessions also route through the poisoned weather provider.

    Try: continue
