SANDBOX READY
EDUCATIONAL SIMULATION. Nothing here touches your real machine. All commands, network calls, and file writes are simulated in your browser. Domains like attacker.example are non-routable.

Walkthrough

  1. 1Agent browses

    Ask the Claude Code agent to fetch and summarize a research article hosted on localhost:8080.

    An entirely benign request. The vulnerability isn't in the prompt — it's in what the page can hide.
    Try: claude "fetch http://localhost:8080/article.html and summarize it"
  2. 2Hidden payload

    The page looks normal in a browser — but it carries adversarial instructions in three delivery vehicles: an HTML comment, a display:none div, and white-on-white text.

    Three vectors in one page. Defense has to catch all of them. An attacker only needs one to land.
    Try: reveal payload
  3. 3Injection fires

    The agent's WebFetch tool returns the full HTML — comments and all. The LLM treats the hidden text as instructions.

    Notice the model accepts a "SYSTEM NOTICE" from a webpage as if it were a real system prompt. There's no privileged channel.
    Try: continue
  4. 4Exfiltration

    The agent runs env > audit-log.txt, base64-encodes it, and ships the secrets to an attacker URL.

    Real-looking secrets — API keys, AWS creds, DB URLs — shipped off without ever asking the user.
    Try: continue
  5. 5Persistence

    The payload appends a backdoor instruction to CLAUDE.md. Every future session in this directory is compromised.

    One fetched page. Every Claude Code session in this directory now starts compromised — the vulnerability outlives the conversation.
    Try: continue
